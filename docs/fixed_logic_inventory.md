# 固定ロジック一覧（重みで置き換え可能な候補）

重み (weights) ではなく、コード内の固定条件・優先度で決まっている箇所を一覧にした。必要に応じて重みで制御できるように変更できる。

---

## game/turn.py

### 1. ボール系グッズの試す順

- **箇所**: 642 行目付近
- **内容**: ポケモンキャッチャーを常に他ボールより先に試す  
  `sort(key=(0 if pokemonkixyatchixya else 1, idx))`
- **重み化案**: `w_goods_use` だけでソート（キャッチャーを先にしたいなら重みを高くする）

### 2. ハイパーボールを turn_count < 2 で試さない

- **箇所**: 339, 640, 786 行目付近
- **内容**: 先行1ターン目など進化不可のターンではハイパーボールを候補から除外
- **重み化案**: ルールとして残すか、または「進化可能ターンでないときは w_goods_use に負のバイアス」などで表現するか検討

### 3. 手札刷新系サポートがあるときだけ「その前にグッズ 1 回」

- **箇所**: 677–685 行目付近
- **内容**: `has_hand_refresh_support`（たんぱんこぞう・博士の研究・ジム・キハダのいずれかが手札にある）のときだけ `_try_goods_before_hand_refresh` を実行
- **重み化案**: このブロックの有無や「いつグッズを先に試すか」を重みで決めるのは難しめ。固定のままにするか、新重みで「このターンでグッズを先に試す」を 0/1 的に決めるか検討

### 4. いれかえ・にげるの「いつ使うか」

- **箇所**: 709–726（いれかえ）, 734–771（にげる）
- **内容**: 「would_be_koed and not can_ko_opponent」「retreat_helps」など、生存・反撃可能性でブロック全体の実行有無を決定
- **重み化案**: ルールとして残すのが自然。どのポケモンを選ぶかは既に `w_swap_target` / `w_retreat_target` で重み済み

### 5. いれかえ先のスコア（生存 >> HP >> 重み）

- **箇所**: 714–718 行目付近
- **内容**: `(1 if survives else 0) * _SWAP_SURVIVE_SCALE + bp.hp + get_swap_target_weight(...)`  
  生存できるかで 10000 倍の差をつけてから HP、その上で重みで補正
- **重み化案**: 重みを主にするなら `score = get_swap_target_weight(...)` を主キーにし、同点時だけ生存・HP を使う（または重みのスケールを大きくして「生存したい候補」に正の重みを付ける）

### 6. にげる先のスコア（KO >> ダメージ >> HP >> 重み）

- **箇所**: 741–765 行目付近
- **内容**: `can_ko * 1_000_000 + dmg * 1000 + hp + get_retreat_target_weight(...)`  
  相手を倒せる > 有効ダメージ > HP の固定優先度。生存できるベンチがいる場合はさらに (hp, attached_energy, weight) でソート
- **重み化案**: 順序を「重み優先」に変えると、逆に「逃がしたいポケモン」を重みで後ろにできる。現状は「戦術優先＋重みで微調整」なので、重みを主にするかは方針次第

### 7. 生贄候補の固定（カラミンゴ名義）

- **箇所**: 170, 174–185 行目付近
- **内容**: `_SACRIFICE_POKEMON_NAMES = frozenset({"カラミンゴ"})`。この名前のベンチがいれば無条件で生贄候補 1 位。いなければ「相手を倒せない・HP 低い」ベンチから選択
- **重み化案**: 廃止して「にげる先」と同様のスコア（例: -hp や「相手を倒せない」ペナルティ）＋ `w_retreat_target` のみで選ぶ。または「生贄にしたい card_id」用の重みを別で用意する

### 8. ポケモンキャッチャーで引き出す相手ベンチのスコア

- **箇所**: 146–166 行目付近 `_best_opponent_bench_index_for_catcher`
- **内容**: `retreat * 1000 + min_energy + get_catcher_target_weight(...)`  
  「逃げにくい・技が遅い」を固定で優先し、重みで補正
- **重み化案**: ソートを「重みのみ」にすると、誰を引き出すかは完全に `w_catcher_target` で制御可能

### 9. エネルギー付与の「進化に必要ならバトル場に付与」

- **箇所**: 391–394 行目付近
- **内容**: `_should_attach_for_evolution(p)` が True のときはエネルギーを必ずバトル場に付与（重みを経由しない）
- **重み化案**: この特別扱いをやめ、付与先候補にバトル場を含めて重みで選ぶ。進化したい場合はそのポケモンに正の `w_energy_attach` を付けておく

## game/attack.py

### 10. 技選択の優先度（KO >> 有効ダメージ >> 重み）

- **箇所**: 379–426 行目付近 `_choose_best_attack_index`
- **内容**: 相手をきぜつさせられる技に `_KO_BONUS_FOR_ATTACK`（10000）を加算。その上で有効ダメージ＋重み。また「しっぺがえし」系・「アベンジナックル」系の特別ボーナスが固定
- **重み化案**: スコアを「重みを主、同点時だけダメージ・KO」にすると、技の選好を重みに寄せられる。KO ボーナスや技別ボーナスは重みに吸収するか、重みのスケールを大きくして再現するか検討

### 11. 技別固定ボーナス

- **箇所**: 396–404 行目付近
- **内容**: ついげきバリバリの「前ターンしびれはり」条件、しっぺがえしのサイド 1 枚ボーナス、アベンジナックルの前ターンきぜつボーナスなど
- **重み化案**: これらの効果は「その技を選んだときの有利さ」なので、`w_attack` で正の重みを付けて表現することは可能。固定ロジックを消して重みに寄せるかは好み

## game/state.py

### 12. きぜつ時の繰り出し（can_ko >> HP >> 重み）

- **箇所**: 391–418 行目付近 `_promote_from_bench`
- **内容**: 「次のターンで相手をきぜつさせられる」ポケモンに `_PROMOTE_CAN_KO_SCALE`（10000）を加算。否則 HP をベースにし、`get_promote_weight` で補正
- **重み化案**: 繰り出しも「重みを主、同点時だけ can_ko / HP」にすると、誰を前に出すかは `w_promote` で制御しやすい

## game/trainers.py

### 13. ハイパーボールで取るポケモンの優先度

- **箇所**: 38–52 行目付近 `_find_pokemon_for_haipaboru`
- **内容**: 山札から (1) 進化可能なポケモン（バトル場→ベンチの順で探す）(2) それ以外のポケモン、の順で 1 枚だけ選ぶ
- **重み化案**: 山札のポケモンに「取ってきたい度」の重みを付けるのは難しい（card_id は手札にないので学習も難しい）。進化優先は定石として残し、必要なら「進化 vs 基本」の優先を重みで切り替える新キーを検討

## まとめ

| 番号 | 内容 | 重み化のしやすさ |
|------|------|------------------|
| 1 | ボール系でキャッチャー優先 | 易（w_goods_use のみでソート） |
| 2 | ハイパーボールを 1 ターン目で使わない | 要検討（ルールのままか、重みで抑えるか） |
| 3 | 手札刷新サポートがあるときだけグッズ先試行 | 難（ブロック単位の制御） |
| 4 | いれかえ・にげるの「使う条件」 | ルールとして維持が無難 |
| 5 | いれかえ先の生存 >> HP >> 重み | 可（重みを主にできる） |
| 6 | にげる先の KO >> ダメージ >> HP >> 重み | 可（重みを主にできる） |
| 7 | 生贄ポケモン名（カラミンゴ）固定 | 可（廃止して重み or 汎用スコアで選ぶ） |
| 8 | キャッチャー対象の「逃げにくい・技遅い」優先 | 易（w_catcher_target のみで選ぶ） |
| 9 | 進化に必要ならエネルギーをバトル場に | 可（特別扱いをやめて重みで選ぶ） |
| 10 | 技選択の KO >> ダメージ >> 重み | 可（重みを主にできる） |
| 11 | 技別ボーナス（しっぺがえし等） | 可（重みに寄せる） |
| 12 | 繰り出しの can_ko >> HP >> 重み | 可（重みを主にできる） |
| 13 | ハイパーボールで進化優先 | 難（山札のカードに重みを付ける仕組みが別途必要） |

「重みを主にする」にすると、現状の「戦術優先＋重みで微調整」から「重みでほぼ全部決める」になり、学習データの質や重みの更新頻度が効きやすくなる。どこまで重みに寄せるかはデッキ・運用方針で決めるとよい。
